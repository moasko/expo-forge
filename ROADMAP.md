# 🗺️ Expo Forge - Strategic Roadmap

This document outlines the planned future features and improvements for **Expo Forge**.

## 🎯 Phase 1: Core Extensibility (Next)

- [ ] **Config File Support**: Allow `forge.config.json` or `forge.config.js` to override default templates and structures.
- [ ] **Custom Templates**: Support for local or remote templates folders.
- [ ] **Interactive CLI**: Add prompts (inquirer/enquirer) for better project configuration during `init`.

## 🔐 Phase 2: Enterprise Modules

- [ ] **Authentication Boilerplate**: Ready-to-use Clerk, Supabase, or Firebase integration templates.
- [ ] **Deep Linking**: Built-in configuration for standard deep linking patterns.
- [ ] **Forms Engine**: Integrated React Hook Form + Zod boilerplate for all features.

## 🧪 Phase 3: Testing & Quality

- [ ] **Jest Setup**: Automatic generation of test files for features.
- [ ] **Storybook Integration**: Optional Storybook setup for component-driven development.
- [ ] **Sentry integration**: One-line configuration for error tracking.

## 🚀 Phase 4: Developer Experience (DX)

- [ ] **Forge UI**: A local web dashboard to manage features and dependencies visually.
- [ ] **VS Code Extension**: Snippets and commands integrated directly into the editor.
- [ ] **CI/CD Templates**: Pre-configured Github Actions for Expo EAS Build and Submit.

## 📱 Phase 5: Advanced Features

- [ ] **Database integration**: Prisma + SQLite/PostgreSQL support.
- [ ] **PWA Support**: First-class support for Progressive Web App features in Expo.
- [ ] **Offline First**: Templates for TanStack Query offline persist and persistence logic.
